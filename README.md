# FreashBite

- Contexte du projet
Vous êtes développeur Frontend chez WebSpark Studio, une agence spécialisée dans la création de solutions web pour les petites entreprises et commerçants locaux.

Votre client, FreshBite, est un restaurant healthy moderne situé à Rabat.

Le restaurant souhaite améliorer son image de marque et attirer de nouveaux clients grâce à un site vitrine simple, clair et professionnel.

Le site doit permettre aux visiteurs :

De découvrir l’identité et le concept du restaurant
De consulter les plats proposés
De visualiser des images de l’ambiance et des menus
De contacter le restaurant via un formulaire fonctionnel avec validation JavaScript
Votre rôle est de réaliser l’intégration responsive du site en HTML5 / CSS3 / JavaScript Vanilla (ES6+).

Pages et Sections à réaliser

1. Page d’Accueil (Home)

    La page d’accueil doit contenir :

    Une bannière principale avec une image représentative + un slogan
    Une courte présentation du restaurant (texte simple)
    Un bouton permettant de descendre vers le menu ou la galerie
    Un header avec navigation (menu) → le menu doit être responsive
2. Page Menu

    Cette page présente les plats proposés. Elle doit contenir :

    Une liste ou des cartes présentant les plats (nom + image + prix)
    Organisation visuelle simple et lisible
    Éventuellement catégories (ex: Petit déjeuner / Déjeuner / Boissons)
3. Page Galerie

    Une galerie d’images du restaurant ou de plats
    Mise en page en grille ou slider simple
    Interaction optionnelle : léger hover ou zoom
4. Page Contact

    Cette page doit contenir :

    Un formulaire de contact avec les champs :
    Nom
    Email
    Message
    Un bouton d’envoi
    Un Google Maps pour afficher la localisation du restaurant
    Validation du formulaire (obligatoire en JavaScript)
    Si un champ est vide → afficher un message d’erreur visible
    Si l’adresse email n’est pas valide → afficher un message d’erreur
    Si tous les champs sont corrects → afficher un message de succès et vider les champs
    Aucun envoi réel du formulaire (pas de backend requis).
    Composants communs aux pages

    Navbar : menu simple, lisible et responsive
    Footer : coordonnées du restaurant + réseaux sociaux
